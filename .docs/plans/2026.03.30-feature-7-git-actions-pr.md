# Feature 7: Git Action Buttons & PR Detection -- Implementation Plan

## Overview

Add Push and PR buttons to the top bar that send predefined messages to the agent. Add PR detection in chat messages and a PR API endpoint that queries GitHub. Add a PR badge in the session header.

---

## Step 1: PR Type

**File:** `src/types/pr.ts`

```ts
export interface PullRequest {
  number: number;
  title: string;
  state: "open" | "closed" | "merged";
  url: string;
  createdAt: string;
  headBranch: string;
  baseBranch: string;
}
```

---

## Step 2: PR API Endpoint

**File:** `src/app/api/sessions/[id]/pr/route.ts`

### GET handler
- Read session's `session.json` to get `repoUrl` and `branchName`
- Parse `repoUrl` to extract `owner` and `repo` (e.g. `https://github.com/owner/repo` -> `owner`, `repo`)
- Call GitHub REST API: `GET https://api.github.com/repos/{owner}/{repo}/pulls?head={owner}:{branchName}&state=all`
  - Use `GITHUB_PERSONAL_ACCESS_TOKEN` as Bearer token
- If PRs found, return the most recent one as `{ pr: PullRequest }`
- If none found, return `{ pr: null }`
- 404 if session not found, 502 if GitHub API fails

## Step 3: Update page.tsx -- Git Action Buttons

**Modify:** `src/app/page.tsx`

### Changes
- Replace the spacer `<div className="w-8" />` in the header with two icon buttons (Push and PR)
- Only visible when `activeSessionId` is set
- Push button: sends `"Push all committed changes to the remote repository on the current branch. Use git push."` via `POST /api/sessions/[id]/message`
- PR button: sends `"Create a pull request from the current branch to main. Use gh pr create with a descriptive title and body based on the changes made."` via `POST /api/sessions/[id]/message`

### PR badge in header
- On session switch + every 60s, fetch `GET /api/sessions/[id]/pr`
- If a PR exists, show `PR #42` pill badge next to branch name (green, links to GitHub in new tab)

## Step 5: PR Detection in ChatView

**Modify:** `src/components/chat-view.tsx`

In `AssistantBubble`:
- After rendering content, scan `message.content` for GitHub PR URL pattern: `https://github.com/[^/]+/[^/]+/pull/\d+`
- If found, render a `<PrCard />` below the message content (fetch PR details from the API)
- Use a simple `usePrDetection` hook that fetches PR info when a PR URL is detected
