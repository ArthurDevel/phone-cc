# Feature 10: Error Handling -- Implementation Plan

## Overview

Add resilient error handling: network loss banner, SSE reconnection with backoff, failed message retry, toast notifications, and agent crash detection. Much of the backend error handling (clone failure, session limit) already exists -- this is primarily frontend work.

---

## Step 1: Toast Notification System

**File:** `src/components/toast.tsx`

Simple toast component:
- Toast context with `addToast(message, variant)` method
- Variants: `error` (red), `warning` (orange), `success` (green)
- Auto-dismiss after 5 seconds
- Stack vertically at the top of the screen
- Wrap in providers.tsx

---

## Step 2: Network Loss Detection

**Modify:** `src/app/page.tsx`

- Listen to `window` online/offline events
- Show yellow banner below header when offline
- Hide when back online

---

## Step 3: SSE Reconnection with Backoff

**Modify:** `src/components/chat-view.tsx` (`useChatStream`)

- On EventSource `onerror`, close and retry with exponential backoff (1s, 2s, 4s, 8s, max 30s)
- On reconnect success, re-fetch message history to catch up
- Show "Reconnecting..." text in chat during retry

---

## Step 4: Failed Message Send + Retry

**Modify:** `src/components/chat-view.tsx`

- Track send errors per message
- On failed send, show "Failed to send" label + "Retry" button on the user bubble
- Retry re-sends the same text

---

## Step 5: Agent Error Card

**Modify:** `src/components/chat-view.tsx`

- When status changes to `error`, show an error card in chat: "Agent disconnected unexpectedly" + "Reconnect" button
- Reconnect button calls `POST /api/sessions/[id]/reconnect`

---

## Step 6: Wire Toast into Sidebar

**Modify:** `src/components/sidebar.tsx`

- Show toast on clone failure (500 from POST /api/sessions)
- Show toast on session limit exceeded (409)

---

## Step 7: Build Verification

- `pnpm build` passes
