# Feature 4: Session List UI (Hamburger Sidebar) — Implementation Plan

## Overview

Build a sliding sidebar with session list, new session creation, and settings navigation. Add a SessionContext for state management. Convert `page.tsx` to a client component that uses the context.

---

## Step 1: SessionContext Provider

**File:** `src/contexts/session-context.tsx`

`"use client"` — React context for global session state.

### State
```ts
interface SessionContextValue {
  sessions: Session[];
  activeSessionId: string | null;
  unreadSessions: Set<string>;
  sidebarOpen: boolean;
  loading: boolean;
  setSidebarOpen: (open: boolean) => void;
  switchSession: (id: string) => void;
  addSession: (session: Session) => void;
  removeSession: (id: string) => void;
  refreshSessions: () => Promise<void>;
}
```

### Behavior
- On mount: fetch `GET /api/sessions`, populate `sessions` array
- `switchSession(id)`: set `activeSessionId`, remove from `unreadSessions`
- `addSession(session)`: prepend to `sessions`, set as active
- `removeSession(id)`: remove from `sessions`. If it was active, set `activeSessionId` to null.
- `setSidebarOpen`: toggle sidebar visibility

### Provider wrapping
Add `<SessionProvider>` in `layout.tsx` wrapping `{children}` inside the `<body>`. Since `layout.tsx` is a server component, create a separate `Providers` client component that wraps the context.

## Step 3: Sidebar Component

**File:** `src/components/sidebar.tsx`

`"use client"` component.

### Structure
```
┌─────────────────────────────────┐
│ [Backdrop: bg-black/50]         │
│ ┌───────────────┐               │
│ │ PhoneCC    [X] │ ← Header     │
│ │                │               │
│ │ [+ New Session]│ ← Button     │
│ │                │               │
│ │ ● tokyo       │ ← Session    │
│ │   My API    [x]│   rows       │
│ │                │               │
│ │ ● paris       │               │
│ │   Frontend  [x]│               │
│ │                │               │
│ │ ─────────────  │ ← Divider    │
│ │ ⚙ Settings    │ ← Link       │
│ └───────────────┘               │
└─────────────────────────────────┘
```

### Overlay & Animation
- Fixed overlay: `fixed inset-0 z-50`
- Backdrop: `bg-black/50`, click to close
- Sidebar panel: `w-72 h-full bg-surface flex flex-col`
- Transition: use CSS `transition-transform duration-200 ease-out`. When closed: `-translate-x-full`. When open: `translate-x-0`.
- Render always but visually hidden when closed (for transition). Use `pointer-events-none` on outer when closed, `pointer-events-auto` on sidebar panel.

### "New Session" Button
- Fetch `GET /api/projects` when the button is tapped (or on sidebar open)
- If no projects: gray button, "Link a project in Settings first."
- If 5 sessions: gray button, "Max 5 sessions. Close one to start new."
- Otherwise: show a dropdown/list of projects. On project tap:
  1. Call `POST /api/sessions` with `{ projectId }`
  2. On success: `addSession(session)`, close sidebar
  3. Show "Creating..." state while in progress

### Session Rows
Each row:
- Status dot: 8px circle. Color based on status (for now just green for active, gray for disconnected — detailed status indicators come in Feature 9).
- Branch name: `font-medium text-sm text-foreground`
- Project name: `text-xs text-muted`
- Close (X) button on right:
  - Call `DELETE /api/sessions/[id]`
  - If `requiresConfirmation: true` in response: `window.confirm("This session has N unpushed commit(s)...")`
  - If confirmed: call DELETE again with `{ confirmed: true }`
  - On success: `removeSession(id)`
- Row click (not X): `switchSession(id)`, close sidebar

### Bottom Section
- Divider: `border-t border-border`
- Settings link: `<Link href="/settings">` with gear icon and "Settings" text
