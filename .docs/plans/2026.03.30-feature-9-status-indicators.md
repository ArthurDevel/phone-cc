# Feature 9: Session Status Indicators -- Implementation Plan

## Overview

Add per-session runtime status tracking (idle/thinking/error/disconnected) across sidebar dots, top bar branch color, and chat thinking indicator. Most infrastructure exists -- the backend already emits `status_change` SSE events and ChatView tracks status locally. Need to lift status into SessionContext so sidebar and top bar can react.

---

## Step 1: Add Status Map to SessionContext

**Modify:** `src/contexts/session-context.tsx`

- Add `statusBySession: Record<string, string>` state
- Export `setSessionStatus(id, status)` method
- Expose `getSessionStatus(id)` or just the map

---

## Step 2: Propagate Status from ChatView

**Modify:** `src/components/chat-view.tsx`

- When `useChatStream` receives a `status_change` SSE event, also call `setSessionStatus` on the context
- This connects the per-session SSE status to the global context

---

## Step 3: Update Sidebar Status Dots

**Modify:** `src/components/sidebar.tsx`

- Use `statusBySession` from context instead of just `session.status`
- Green = idle, Orange pulsing = thinking, Red = error, Gray = disconnected

---

## Step 4: Update Top Bar Branch Color

**Modify:** `src/app/page.tsx`

- Color the branch name based on active session's runtime status
