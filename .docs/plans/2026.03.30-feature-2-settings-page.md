# Feature 2: Settings Page — Implementation Plan

## Overview

Build a settings page at `/settings` for managing linked GitHub projects, with CRUD API routes and a filesystem-backed JSON store at `~/.phonecc/projects.json`.

---

## Step 1: Project Type Definition

**File:** `src/types/project.ts`

```ts
export interface Project {
  id: string;
  name: string;
  repoUrl: string;
  defaultBranch: string;
}
```

---

## Step 2: Projects Storage Helper

**File:** `src/lib/projects.ts`

Filesystem helper for reading/writing `~/.phonecc/projects.json`.

- `getProjectsFilePath()` → returns `path.join(os.homedir(), '.phonecc', 'projects.json')`
- `ensurePhoneccDir()` → creates `~/.phonecc/` if it doesn't exist (`fs.mkdirSync` with `recursive: true`)
- `readProjects(): Promise<Project[]>` → reads and parses the JSON file. If file doesn't exist, return `[]`.
- `writeProjects(projects: Project[]): Promise<void>` → writes the array to the JSON file (pretty-printed).
- `addProject(data: { name: string; repoUrl: string; defaultBranch: string }): Promise<Project>` → generates UUID via `crypto.randomUUID()`, appends to the list, writes, returns the new project.
- `removeProject(id: string): Promise<boolean>` → filters out by ID, writes. Returns `false` if ID was not found.

---

## Step 3: API Route — GET/POST `/api/projects`

**File:** `src/app/api/projects/route.ts`

### GET handler
- Call `readProjects()`
- Return `Response.json({ projects })`

### POST handler
- Parse request body: `{ name, repoUrl, defaultBranch }`
- Validate: all three fields must be non-empty strings. Return 400 if invalid.
- Call `addProject({ name, repoUrl, defaultBranch })`
- Return `Response.json(project)` with status 201

---

## Step 4: API Route — DELETE `/api/projects/[id]`

**File:** `src/app/api/projects/[id]/route.ts`

### DELETE handler
- Extract `id` from `await ctx.params` (Next.js 16 pattern: `ctx: RouteContext<'/api/projects/[id]'>`)
- Call `removeProject(id)`
- If returns `false`, return 404 `{ error: "Project not found" }`
- Otherwise return `Response.json({ success: true })`

## Step 5: Settings Page UI

**File:** `src/app/settings/page.tsx`

Client component (`"use client"`) with the following structure:

### Layout
```
┌────────────────────────────┐
│ ← Back          Settings   │  ← Header (h-12, border-b)
├────────────────────────────┤
│                            │
│  Linked Projects           │  ← Section heading
│                            │
│  ┌──────────────────────┐  │
│  │ My API               │  │  ← Project row
│  │ github.com/u/repo  main │ │
│  │                    ✕  │  │
│  └──────────────────────┘  │
│                            │
│  (or: "No projects linked  │  ← Empty state
│   yet. Add one below.")    │
│                            │
│  ── Add Project ─────────  │  ← Form section
│  Name:      [__________]   │
│  Repo URL:  [__________]   │
│  Branch:    [__main____]   │
│  [  Add  ]                 │
│                            │
└────────────────────────────┘
```

### State
- `projects: Project[]` — fetched from `GET /api/projects` on mount
- `loading: boolean` — true while fetching
- `name`, `repoUrl`, `defaultBranch` — form state (defaultBranch defaults to `"main"`)
- `submitting: boolean` — true while adding

### Behavior
- **On mount:** fetch `GET /api/projects`, populate list
- **Add:** POST to `/api/projects`, on success prepend to list, clear form
- **Delete:** show `window.confirm()` dialog. If confirmed, DELETE to `/api/projects/[id]`, remove from local state.
- **Back arrow:** `router.push("/")` or `<Link href="/">`

### Styling
- Header: same height and border style as main page top bar
- Back arrow: SVG chevron-left icon, white, 20x20
- Title "Settings": centered, `text-sm font-medium`
- Section heading: `text-xs uppercase tracking-wider text-muted mb-3`
- Project rows: `bg-surface rounded-lg p-3 mb-2`, with:
  - Name: `font-medium text-sm`
  - Repo URL: `text-xs text-muted truncate`
  - Branch: small pill badge `bg-accent/20 text-accent text-xs px-2 py-0.5 rounded-full`
  - Delete button: `text-danger hover:text-danger/80`, X icon
- Form inputs: same style as the message input (`bg-surface border border-border rounded-lg h-10 px-3 text-sm`)
- Add button: `bg-accent hover:bg-accent-hover text-white rounded-lg h-10 px-4 text-sm font-medium w-full`
- Empty state: `text-muted text-sm text-center py-8`

## Step 7: API Verification (curl)

```bash
# POST — create a project
curl -X POST http://localhost:3000/api/projects \
  -H "Content-Type: application/json" \
  -d '{"name":"Test Project","repoUrl":"https://github.com/user/repo","defaultBranch":"main"}'
# Expect: 201, { id, name, repoUrl, defaultBranch }

# GET — list projects
curl http://localhost:3000/api/projects
# Expect: 200, { projects: [{ id, name, repoUrl, defaultBranch }] }

# DELETE — remove project
curl -X DELETE http://localhost:3000/api/projects/{id}
# Expect: 200, { success: true }

# DELETE again (should 404)
curl -X DELETE http://localhost:3000/api/projects/{id}
# Expect: 404, { error: "Project not found" }
```
