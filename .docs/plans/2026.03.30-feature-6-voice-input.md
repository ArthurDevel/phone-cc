# Feature 6: Voice Input (Deepgram) -- Implementation Plan

## Overview

Add push-to-talk voice input using Deepgram speech-to-text. Since Next.js 16 does not support WebSocket endpoints, we use a standalone WebSocket server (Hono + @hono/node-server, both already installed) running on port 3001. The client connects via WebSocket to stream audio; the server proxies to Deepgram's API (keeping the API key server-side). Also: make the text input functional, add a cancel endpoint, and wire up the mic button.

## Step 2: WebSocket Proxy Server

**File:** `src/server/deepgram-ws.ts`

A standalone Hono server on port 3001 with a WebSocket endpoint at `/deepgram`.

### Flow
1. Client connects to `ws://localhost:3001/deepgram`
2. Server opens a WebSocket to Deepgram: `wss://api.deepgram.com/v1/listen?model=nova-2&language=en&endpointing=1500&interim_results=true&punctuate=true`
3. Relay: client audio chunks -> Deepgram, Deepgram transcription events -> client
4. On client disconnect: close Deepgram connection
5. On Deepgram disconnect/error: send error event to client, close

### Deepgram auth
Use `DEEPGRAM_API_KEY` from `.env.local` (loaded via dotenv).

### Message format (server -> client)
```json
{ "type": "transcript", "text": "hello world", "is_final": true, "speech_final": false }
{ "type": "error", "message": "Deepgram connection failed" }
```

### Start script
Add to `package.json`:
```json
"scripts": {
  "dev:ws": "tsx src/server/deepgram-ws.ts",
  "dev:all": "concurrently \"pnpm dev\" \"pnpm dev:ws\""
}
```

Install `concurrently` as a dev dependency. Also need `tsx` (check if already available) and `dotenv`.

## Step 4: Update ChatView -- Mic Button & Input

**Modify:** `src/components/chat-view.tsx`

### Text input (already functional from Feature 5)
The text input with Enter-to-send is already working. No changes needed.

### Mic button
Add a mic button next to the send button in the input bar.

#### States
- **Idle:** mic icon, `bg-accent` (indigo)
- **Recording:** mic icon, `bg-danger` (red), with pulsing ring animation

#### Push-to-talk behavior
- `onPointerDown`: start recording
  1. If session status is `thinking`, call `POST /api/sessions/[id]/cancel` first
  2. Request mic permission: `navigator.mediaDevices.getUserMedia({ audio: true })`
  3. Open WebSocket to `ws://localhost:3001/deepgram`
  4. Create `MediaRecorder` or use `AudioContext` + `ScriptProcessorNode` to get raw audio chunks
  5. Stream audio chunks to WebSocket as binary data
  6. Receive transcription events, display interim text as live preview above input bar
  7. On `speech_final: true` (endpointing): accumulate final transcript, send as message, clear preview, keep recording

- `onPointerUp` / `onPointerLeave`: stop recording
  1. Close WebSocket
  2. Stop MediaStream tracks
  3. If accumulated transcript exists, send as message
  4. Clear live preview

#### Audio format
Deepgram accepts raw PCM or WebM/Opus. Simplest: use `MediaRecorder` with `audio/webm;codecs=opus` (widely supported). Send each `dataavailable` chunk directly to the WebSocket.

#### Live preview
Show interim transcript text above the input bar in a small overlay:
- `absolute bottom-full left-0 right-0 px-4 py-2 text-sm text-muted italic`
- Updates as interim results arrive
- Clears when recording stops or message is sent
